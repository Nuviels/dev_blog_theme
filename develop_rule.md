# Development Rules

## Language Sync Rule (Required)

- For this platform, every user-facing change must be reflected in both `kr` and `en`.
- When changing pages or routing, update both `/kr/*` and `/en/*`.
- When changing copy, notices, or settings, update both Korean and English versions together.
- Do not publish or modify content for only one language (`kr` only or `en` only).

## Pre-PR / Pre-Deploy Checklist

1. Verify the same feature behavior in both `/kr/` and `/en/`.
2. Verify there are no missing pages in either language scope.
3. Verify links stay in the same language scope (no unintended cross-language jump).
