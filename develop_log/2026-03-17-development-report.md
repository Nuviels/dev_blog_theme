# 개발 보고서 - 2026-03-17

## 1) 환경/빌드 안정화

- `Gemfile`의 Bundler 플랫폼 파싱 이슈를 수정했습니다.
- `.git` 메타데이터가 없는 로컬 복사본에서도 동작하도록 `jekyll-theme-chirpy.gemspec`에 fallback 로직을 추가했습니다.
- `rbenv` 기반으로 Ruby `3.3.10` 환경을 구성했습니다.
- 시스템 Ruby/Bundler 대신 rbenv shim 기반 실행으로 전환했습니다.
- 번들 설치 경로를 `vendor/bundle`로 설정하고 gem 설치를 완료했습니다.
- `npm install`, `npm run build`를 실행해 `assets/js/dist` 누락 파일을 생성했습니다.

## 2) 비(非) git 환경 대응

- `_plugins/posts-lastmod-hook.rb`를 수정해 git 컨텍스트가 없을 때 git 의존 로직을 건너뛰도록 했습니다.
- `.git` 없는 폴더에서 반복적으로 발생하던 git fatal 로그를 제거했습니다.

## 3) 다국어 URL 구조(kr/en) 구축

- 언어 분리 진입 페이지를 추가했습니다.
  - `/kr/`
  - `/en/`
- 루트 `/`는 `/kr/`로 리다이렉트되도록 구성했습니다.
- `kr/en` 각각에 대해 홈/탭 페이지를 분리 구성했습니다.
- `_config.yml` 기본 permalink를 언어 prefix 구조로 변경했습니다.
  - `kr`: `/kr/posts/:title/`
  - `en`: `/en/posts/:title/`
- 포스트 작성 디렉터리 규칙을 추가했습니다.
  - `_posts/kr`
  - `_posts/en`

## 4) 언어별 콘텐츠 필터링 적용

`post_lang` 기준으로 주요 화면에서 `kr/en`이 섞이지 않도록 필터링을 적용했습니다.

- 홈 목록
- 아카이브
- 태그/태그 상세
- 카테고리/카테고리 상세
- 우측 최근글 패널
- 사이드바/탑바 홈 라우팅

## 5) 카테고리 시스템 확장

- 계층형 카테고리 트리 생성 플러그인을 구현했습니다.
  - 예: `NSHC > 2026 > ADD`
- 카테고리 트리 렌더링 include와 카테고리별 포스트 섹션 렌더링 include를 추가했습니다.
- 숫자 카테고리(예: `2026`)에서 `slugify` 오류가 나지 않도록 문자열 정규화 처리를 적용했습니다.

## 6) 가시성 규칙(`hidden`, `local_only`) 적용

- 플러그인/템플릿에 공통 가시성 규칙을 반영했습니다.
  - `hidden: true` → 목록성 화면에서 숨김
  - `local_only: true` → 개발 환경에서만 노출, production에서 제외
- 목록/카테고리/태그/아카이브 전반에 일관되게 적용했습니다.

## 7) Categories UX 개선 이력

카테고리 페이지 흐름을 단계적으로 개선했습니다.

- 2패널 동시 노출 구조에서 단일 흐름 구조로 변경
  - 최초: `Category Tree`만 노출
  - 트리 클릭: 해당 `Posts By Category` 화면으로 전환
  - `Back to Tree`: 트리 화면 복귀
- 트리/포스트 섹션 접기/펼치기 제어 추가
- 긴 목록 대응을 위한 내부 스크롤 추가
- 토글 UI를 둔탁한 버튼형에서 chevron 아이콘형으로 변경
